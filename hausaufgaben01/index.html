Що таке github?

Це:

  1. Один з найбільших вебсервісів для спільної розробки програмного забезпечення.
  2. Безкоштовний сервіс для проєктів з відкритим вихідним кодом, з наданням користувачам усіх своїх можливостей
  3. Можливість контролювати різні версії кода, управляти ісходним кодом та працювати разом з командою.

Перелік основних команд, якi потрібно знати:

  • "git config --global user.name" - ім'я користувача
  • "git config --global user.email" - адреса електронної пошти
  • "git init" - створює новий GIT репозиторій
  • "git clone" - копіювання існуючого сховища з віддаленого джерела
  • "git config --global core.autocrlf true"; git config --global core.safecrlf false" - налаштування методу закінчення рядків
  • "git status" - команда, яка створює новий репозиторій; перевірка статусу файлів
  • "git config --list" - перегляд своїх налаштувань
  • "git add ." - приймає шлях файлу або директорії, додає усі файли в цій директорії
  • "git diff" - відображає що саме змінили, а не просто файли які змінили
  • "git commit" - записує знімок, який створено в індексі
  • "git rm" - видаляє файл з Git, якщо треба прибрати його з контрольованих файлів (з індексу) та створити коміт, а також видаляє файл з робочої директорії, щоб наступного разу він не відображався неконтрольованим
  • "git push" - поміщає зміни в головну гілку віддаленого сховища, яке пов’язане з робочим каталогом
  • "git branch" - створює та відображає всі гілки у репозиторії
  • "git branch –d <ім’я_гілки>" - видаляє гілку
  • "git merge <ім’я_гілки>" - поєднання гілки в активну
  • "git checkout" - створення гілок або перемикання між гілками репозиторію
  • "git log" - відображає історію коммітів поточного репозиторію
  • "git pull" - автоматично отримує зміни з віддаленого репозиторію
  • "git revert" - створення нового коміту, що скасовує всі зміни з існуючого коміту
  • "git reset" - відміняє зміни, які ще ніхто не отримавабо які ще не зафіксовані
  • "git rebase" - перенесення змін з однієї гілки в іншу
  • "git cherry-pick" - бере зміни, внесені одним комітом, і намагається повторно застосувати їх у вигляді нового коміта в поточній гілці
  • "git stash" - бере чорновий стан робочої директорії (змінені супроводжувані файли та індексовані зміни) та зберігає їх у стеку незавершених змін, які можна знову використати будь-коли

    • Які можливості файлу .gitignore?

    1. дає можливість виключати зайві файли для репозиторія;
    2. дає можливість налаштування ігнорування для любого Git репозиторія в локальній мережі;
    3. дає можливість приховувати файли та папки від системи контролю версій Git.
